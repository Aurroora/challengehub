# Техническое задание: ChallengeHub

## 1. Цель проекта
Разработка веб-сервиса для создания, отслеживания и анализа персональных челленджей. 
Сервис помогает пользователям формировать полезные привычки через систему 
ежедневных заданий с визуализацией прогресса и глубокой аналитикой успешности.

## 2. Роли пользователей
- **Гость**: Просмотр каталога шаблонов челленджей, регистрация, вход в систему
- **Авторизованный пользователь**: Запуск челленджей, ежедневное отметка выполнения, 
  просмотр статистики, создание кастомных челленджей
- **Администратор**: Управление шаблонами, пользователями, просмотр общей статистики

## 3. Модели данных

### ChallengeTemplate (Шаблон челленджа)
- `title` - название челленджа (строка, 200 символов)
- `description` - подробное описание (текст)
- `category` - категория (спорт, творчество, обучение, здоровье, продуктивность)
- `duration_days` - длительность в днях (целое число)
- `difficulty` - сложность (легко/средне/сложно)
- `image` - изображение для челленджа (необязательно)
- `is_active` - активен ли шаблон (булево)
- `created_at` - дата создания (дата-время)

### UserChallenge (Запущенный челлендж пользователя)
- `user` - связь с пользователем (ForeignKey)
- `template` - связь с шаблоном (ForeignKey)
- `start_date` - дата начала (дата)
- `end_date` - дата окончания (вычисляемое поле)
- `status` - статус (активен/завершен/провален)
- `current_streak` - текущая серия выполненных дней (целое число)
- `completed_days` - количество выполненных дней (целое число)

### DailyCheckin (Ежедневная отметка)
- `user_challenge` - связь с запущенным челленджем (ForeignKey)
- `date` - дата отметки (дата)
- `is_completed` - выполнено ли задание (булево)
- `notes` - заметки пользователя (текст, необязательно)
- `rating` - самооценка дня (1-5, необязательно)

## 4. Ключевой функционал (User Stories)
1. **Просмотр и выбор**: Пользователь просматривает каталог шаблонов челленджей, 
   фильтрует по категориям и сложности, выбирает подходящий.
2. **Запуск и отслеживание**: Пользователь запускает челлендж, ежедневно отмечает 
   выполнение, видит прогресс в виде шкалы и календаря.
3. **Аналитика прогресса**: Система автоматически рассчитывает процент выполнения, 
   оставшиеся дни, строит графики прогресса и предоставляет статистику успешности.
4. **Создание своих челленджей**: Пользователь может создать полностью кастомный 
   челлендж без использования шаблонов.
5. **Анализ истории**: Пользователь видит статистику по всем пройденным челленджам: 
   процент завершенных, средняя длительность, успешность по категориям.

## 5. Внешние интеграции / Аналитика
**Аналитика**: 
- Использование библиотеки Matplotlib/Plotly для построения графиков прогресса
- Расчет сложной статистики: процент завершения, анализ паттернов выполнения 
  (в какие дни чаще пропускаются задания), рекомендации на основе истории
- Pandas для агрегации и анализа данных пользователей

**API**: На первом этапе внешние API не используются, фокус на глубокой аналитике 
внутренних данных.
