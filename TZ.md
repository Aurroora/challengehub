## 1. Цель проекта
Создание веб-сервиса для трекинга персональных челленджей и формирования полезных привычек с интегрированной Data Science аналитикой.

Проект решает проблему мотивации и систематического подхода к достижению личных целей, предоставляя пользователям инструменты для создания, отслеживания и анализа выполнения персональных челленджей. Сервис помогает превращать разовые действия в устойчивые привычки через визуализацию прогресса и мотивационную систему достижений.

## 2. Роли пользователей
**Гость**
- просмотр главной страницы с описанием функционала;
- регистрация и авторизация в системе.

**Авторизованный пользователь**
- создание кастомных челленджей с персонализированными параметрами;
- ежедневные отметки выполнения с оценкой дня и заметками;
- просмотр статистики и аналитики прогресса в виде интерактивных графиков;
- отслеживание и коллекционирование мотивационных достижений;
- управление профилем и историей челленджей.

**Администратор**
- управление всеми моделями данных через административную панель Django;
- просмотр статистики пользователей и системных показателей;
- выполнение массовых операций и администрирование контента.

## 3.  Модели данных (сущности)
### UserChallenge (Челлендж пользователя)
Основная единица системы — пользовательский челлендж с полной кастомизацией.

**Поля:**
- user (ForeignKey — владелец челленджа)
- template (ForeignKey — опциональная ссылка на шаблон)
- custom_title (CharField — уникальное название, до 100 символов)
- custom_description (TextField — детальное описание цели)
- custom_category (CharField — категория: здоровье, обучение, спорт, творчество и т.д.)
- custom_duration (IntegerField — длительность в днях, от 1 до 365)
- custom_difficulty (CharField — уровень сложности: легкий, средний, сложный)
- start_date (DateField — дата начала челленджа)
- status (CharField — статус: активен, завершен, провален)
- current_streak (IntegerField — текущая серия подряд выполненных дней)
- completed_days (IntegerField — общее количество выполненных дней)
- notes (TextField — дополнительные заметки пользователя)
- created_at (DateTimeField — дата создания)

### DailyCheckin (Ежедневная отметка)
Отметка выполнения за конкретный день с оценкой и рефлексией.

**Поля:**
- user_challenge (ForeignKey — связанный челлендж)
- date (DateField — дата отметки)
- is_completed (BooleanField — факт выполнения за день)
- rating (IntegerField — оценка дня от 1 до 5 с эмодзи-отображением)
- notes (TextField — заметки и рефлексия за день)
- created_at (DateTimeField — время создания записи)

### Achievement (Достижение)
Мотивационная сущность для поощрения пользователя за прогресс.

**Поля:**
- user (ForeignKey — владелец достижения)
- type (CharField — тип: серии, завершение, разнообразие, регулярность)
- title (CharField — название достижения)
- description (TextField — описание условия получения)
- icon (CharField — иконка-эмодзи для визуализации)
- earned_date (DateField — дата получения)
- progress (IntegerField — текущий прогресс к цели)
- target (IntegerField — целевое значение для получения)

### ChallengeTemplate (Шаблон челленджа)
Демонстрационная модель с предустановленными челленджами.

**Поля:**
- title (CharField — название шаблона)
- description (TextField — описание)
- category (CharField — категория)
- duration_days (IntegerField — рекомендуемая длительность)
- difficulty (CharField — уровень сложности)
- image (ImageField — иллюстрация)
- is_active (BooleanField — активен ли шаблон)
- created_at (DateTimeField — дата создания)

## 4. Ключевой функционал
**Регистрация и создание челленджей:** Пользователь регистрируется в системе, создает персонализированные челленджи с кастомными названиями, описаниями, категориями и длительностью.
**Ежедневный трекинг:** Пользователь ежедневно отмечает выполнение челленджа, оценивает день по 5-балльной шкале, добавляет заметки и рефлексию.
**Визуализация прогресса:** Система предоставляет календарь выполнения, индикаторы прогресса, графики динамики оценок и выполненых дней.
**Data Science аналитика:** Пользователь просматривает статистику в виде интерактивных графиков:
- Динамика ежедневных оценок (линейные графики Plotly)
- Выполнение по дням (столбчатые диаграммы)
- Распределение челленджей по категориям (круговые диаграммы)
- Сравнение процента выполнения (сравнительные графики)

**Мотивационная система:** Автоматическое создание достижений при выполнении условий (12+ типов достижений), отображение коллекции полученных ачивок с прогрессом.
**Автоматическое управление:** Система автоматически переводит челленджи в статус "завершен" при достижении установленной длительности, подсчитывает серии и статистику.

## 5. Внешние интеграции / Аналитика
**Библиотеки анализа данных**: 
- Plotly 5.17.0 - для построения интерактивных графиков прогресса
- Pandas 2.0.3 - для статистических расчетов и анализа данных пользователей

**Аналитические функции**:
- Реальные расчеты на основе пользовательских данных, а не статичные демо-данные
- Интерактивные графики с возможностью детализации (зум, фильтрация)
- Автоматическое обновление аналитики при добавлении новых данных

## 6. Изменения в ходе реализации
**Расширенная система достижений:** В процессе разработки добавлена прогрессивная система мотивации с 12+ типами достижений вместо изначально запланированных 5. Реализованы:
1. Достижения за серии подряд выполненных дней (3, 7, 30 дней)
2. Достижения за завершение челленджей разной сложности
3. Достижения за разнообразие категорий
4. Достижения за общую активность в системе
**Улучшенный алгоритм аналитики:** Изначально планировалась базовая статистика, но реализована полноценная Data Science интеграция с:
1. Расчетом корреляции между оценками дней и успешностью челленджей
2. Анализом успешности по категориям и сложности
3. Визуализацией трендов и паттернов выполнения
4. Прогрессивными графиками с адаптивным дизайном
**Усовершенствованная архитектура:** Добавлен модуль achievements.py для отдельной логики работы с достижениями, что повысило модульность и поддерживаемость кода. Реализованы кастомные валидаторы для форм и автоматические обработчики сигналов для обновления прогресса.
